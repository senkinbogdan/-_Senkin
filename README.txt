ОПИСАНИЕ ПРОЕКТА
================================================================================

Приложение для автоматического учета рабочего времени сотрудников с 
возможностью визуализации и редактирования данных.

Технологии: Python 3.8+, PyQt6, SQLite3
Архитектура: Модульная структура с разделением интерфейса и логики

================================================================================
СТРУКТУРА ПРОЕКТА
================================================================================

iub.py
   Главный файл приложения
   - Класс MainApp - основное окно
   - Авторизация и регистрация
   - Интеграция с модулем вкладок
   - Управление сессиями пользователей

user_tabs.py
   Модуль функциональных вкладок
   - TimeTrackingTab - учет времени с редактированием
   - UserInfoTab - профиль и статистика пользователя
   - UserTabsWidget - контейнер вкладок
   - Функция color_for_hours - цветовая индикация

styles.py
   Файл стилей Qt CSS
   - ROSNEFT_STYLES - корпоративная цветовая схема
   - Стили для всех элементов интерфейса

interface_1.ui
   Файл интерфейса Qt Designer
   - Описание layout главного окна
   - Виджеты страниц входа, регистрации, главной

users_1.db
   База данных SQLite
   - Таблица users - пользователи
   - Таблица usage_hours - учет времени

rosneft.png
   Логотип компании (150x80px)

ТАБЛИЦА: users
+-------------+----------+---------+----------+
| Поле        | Тип      | Ключ    | Описание |
+-------------+----------+---------+----------+
| id          | INTEGER  | PRIMARY | ID       |
| name        | TEXT     |         | Имя      |
| surname     | TEXT     |         | Фамилия  |
| patronymic  | TEXT     |         | Отчество |
| email       | TEXT     | UNIQUE  | Email    |
| phone       | TEXT     |         | Телефон  |
| password    | TEXT     |         | SHA-256  |
| position    | TEXT     |         | Должность|
+-------------+----------+---------+----------+

ТАБЛИЦА: usage_hours
+----------+---------+---------------+------------------+
| Поле     | Тип     | Ключ          | Описание         |
+----------+---------+---------------+------------------+
| id       | INTEGER | PRIMARY       | ID               |
| user_id  | INTEGER | FOREIGN KEY   | ID пользователя  |
| day      | INTEGER |               | День (0-6)       |
| hours    | REAL    |               | Количество часов |
+----------+---------+---------------+------------------+

CONSTRAINT: UNIQUE(user_id, day) - один пользователь, один день

КЛАСС: MainApp (iub.py)
   Описание: Главное окно приложения
   
   Методы:
   • __init__() - инициализация интерфейса
   • show_login() - страница входа
   • show_register() - страница регистрации
   • show_main() - главная страница
   • do_login() - обработка входа
   • do_register() - обработка регистрации
   • do_logout() - выход из системы
   • _start_user_session(user_id) - старт сессии
   • _on_tick() - обработчик таймера (каждую секунду)
   • _save_session_seconds_to_db() - сохранение времени
   • _setup_logo() - установка логотипа
   • _setup_tabs_on_main_page() - создание вкладок

КЛАСС: TimeTrackingTab (user_tabs.py)
   Описание: Вкладка учета рабочего времени
   
   Методы:

   • set_user(user_id) - установка пользователя
   • update_heatmap() - обновление тепловой карты
   • enable_edit_mode() - включение редактирования
   • disable_edit_mode() - выключение редактирования
   • save_changes() - сохранение изменений в БД
   • cancel_edit() - отмена редактирования

КЛАСС: UserInfoTab (user_tabs.py)
   Описание: Вкладка профиля пользователя
   
   Методы:
   • init_ui() - создание интерфейса
   • set_user(user_id) - установка пользователя
   • load_user_info() - загрузка данных из БД
   • load_statistics() - загрузка статистики
   • update_statistics() - обновление статистики

ФУНКЦИИ:
   • hash_password(password) - SHA-256 хэширование
   • ensure_db() - создание таблиц БД
   • create_empty_usage_for_user(user_id) - инициализация записей
   • color_for_hours(hours) - цвет по количеству часов

================================================================================
ЛОГИКА РАБОТЫ
================================================================================

ПРОЦЕСС АВТОРИЗАЦИИ:
1. Ввод email и пароля
2. Хэширование пароля (SHA-256)
3. Проверка в БД
4. Создание сессии при успехе
5. Запуск таймера учета времени

УЧЕТ РАБОЧЕГО ВРЕМЕНИ:
1. При входе: session_start_time = текущее время
2. Таймер срабатывает каждую секунду: session_seconds++
3. Каждые 10 секунд:
   - Конвертация секунд в часы
   - Добавление к текущему дню
   - Сохранение в БД
   - Обновление интерфейса
4. При выходе: финальное сохранение

РЕДАКТИРОВАНИЕ ВРЕМЕНИ:
1. Сохранение исходных значений
2. Включение режима редактирования ячеек
3. Валидация введенных данных
4. SQL UPDATE для каждого дня
5. Восстановление режима просмотра

================================================================================
БЕЗОПАСНОСТЬ
================================================================================

ХРАНЕНИЕ ПАРОЛЕЙ:
• Алгоритм: SHA-256
• Соль: отсутствует (рекомендуется добавить)
• Необратимое хэширование
• 64 символа hex-строка

SQL INJECTION:
• Все запросы используют параметризацию (?)
• Защита от SQL-инъекций
• Пример: cur.execute("SELECT * FROM users WHERE email=?", (email,))

ВАЛИДАЦИЯ ДАННЫХ:
• Email: проверка наличия "@"
• Пароль: совпадение при регистрации
• Часы: диапазон 0-24, числовой формат
• Уникальность email в БД

================================================================================
ПРОИЗВОДИТЕЛЬНОСТЬ
================================================================================

ОПТИМИЗАЦИИ:
• Автосохранение каждые 10 секунд (не каждую)
• Индексы на email (UNIQUE) для быстрого поиска
• Ленивая загрузка данных (только при открытии вкладки)
• Минимальные перерисовки интерфейса

ПОТРЕБЛЕНИЕ РЕСУРСОВ:
• RAM: ~50-70 МБ
• CPU: <1% (в режиме ожидания)
• Размер БД: ~10 КБ на пользователя

ДОБАВЛЕНИЕ НОВОЙ ВКЛАДКИ:
1. Создать класс наследник QWidget в user_tabs.py
2. Реализовать метод init_ui()
3. Добавить в UserTabsWidget.init_ui():
   self.new_tab = NewTab()
   self.addTab(self.new_tab, "Название")

ДОБАВЛЕНИЕ ПОЛЯ В БД:
1. Создать миграцию:
   ALTER TABLE users ADD COLUMN new_field TEXT;
2. Обновить ensure_db() в iub.py
3. Добавить в формы ввода/вывода

ИЗМЕНЕНИЕ ЦВЕТОВОЙ СХЕМЫ:
1. Отредактировать styles.py
2. Изменить color_for_hours() в user_tabs.py
3. Обновить легенду в TimeTrackingTab
